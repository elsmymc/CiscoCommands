Basic Security Commands

Here you will learn the basic security commands for setting up a Cisco router.

RO(config)#line con 0 RO(config-line)#password password RO(config-line)#login RO(config)#exit

Console Port Access Settings

RO(config)#line vty 0 4 RO(config-line)#password password RO(config)#login

Configure Virtual Terminal Lines (VTYs) to enable remote access to the device, SSH, and Telnet.

Router(config)# line vty 0 4 Router(config-line)# exec-timeout 10

Set up the device to disconnect inactive users after 10 minutes

RO(config)#enable secret password

Encrypted Password (MD5 hash) settings

RO(config)#service password-encryption

Allows you to encrypt the passwords in the configuration file

RO(config)#banner motd X AUTHORIZED ACCESS ONLY X

Displays Notice of the Day message. Any user who tries to log in can see the prompts

RO(config)# login block-for 120 attempts 3 within 60

Configure to block login attempts for 120 seconds if there are three failed login attempts within 60 seconds

RO(config)#ip domain-name domain.com

Configure DNS domain name

RO(config)#crypto key generate rsa general-key modulus 1024

Generate one-way secret keys

RO(config)#username user secret password

Configure username and password

RO(config)#line vty 0 4

Configuring Virtual Terminal Lines (VTY)

RO(config-line)#login local

Verify or Create a Local Database Entry

RO(config-line)#transport input ssh

Enable inbound SSH sessions by VTY

R(config)#ip ssh version 2

Define the SSH version to use

R(config-line)#transport input telnet ssh

Enable the VTY line to use SSH

SW(config-if)#switchport mode Access

Configure Switch Interfaces as Access Port

SW(config-if)#switchport port-security

Configure port security on interfaces

SW(config-if)#switchport port-security maximum 1

Specify the maximum number of MAC addresses allowed

SW(config-if)#switchport port-security violation shutdown

Define the action to take when a violation occurs

SW(config-if)#switchport port-security mac-address

Specifying Allowed MAC Address
