Frame Relay

Here you will learn the frame relay for configuring a Cisco router.

RO(config-if)#encapsulation frame-relay

Enable frame relay encap

RO(config-if)#encapsulation frame-relay ietf

Enable ietf frame relay encapsulation on the interface. Used if Non-CISCO Routers are connected

RO(config-if)#frame-relay interface-dlci 100

Assigning DLCI number 100 on an interface

RO(config-if)#frame-relay map ip 192.168.100.1 110 broadcast

It is used to assign remote IP with local dlci statically. Assignment is done automatically using the reverse arp protocol

RO(config-if)#no frame-relay inverse arp

It is used to disable reverse ARP. If we turn off reverse ARP, we have to statically map remote IP and local DLCI

RO(config-if)#interface serial 0/0/0/0.1 point-to-point

Create a Peer-to-Peer Subinterface Numbered 1

RO#show frame-relay map

Used to view the status of the IP/DLCI map input

RO#show frame-relay pvc

Used to view the status of all configured PVCs

Router(config-if)#ip ospf dead-interval 50

Set the dead interval time to 50 seconds. The dead interval time must match on both routers to become neighbors

RO#show frame-relay lmi

Displays LMI statistics, including types and message exchanged

RO#clear frame-relay counters

Clear all frame relay counters

RO#clear frame-relay inarp

Reset the map table and delete all reverse arp entries

RO#debug frame-relay lmi

Enable debugging processes in LMI

RO#show ip ospf neighbor list all

Displays basic information of neighbors with OSPF

RO#no debug frame-relay lmi

Disable debugging processes in LMI
