Virtual Local Area Network (VLAN)

Here you will learn the basic commands for configuring a virtual local area network (VLAN) on Cisco router.

RVTP (Vlan Trunking Protocol)

Used by CISCO switches to exchange VLAN information. You can synchronize vlan information (for example, vlan ID or VLAN name) with switches within the same vtp domain

Trunk

A trunk can consist of multiple cables, cables, or fiber optic strands grouped together to maximize the bandwidth available on a single physical cable

SW(config)#vtp mode server

Configuring the Switch as a VTP Server

SW(config)#vtp mode client

Configuring the Switch as a VTP Client

SW(config)#vtp mode trasnparent

Configuring the Switch as a Transparent VTP

SW(config)#no vtp mode configure

Configure the default switch in VTP server mode

SW(config)#vtp domain domain-name

Set VTP domain name

SW(config)#vtp password password

Set VTP password. Password is case sensitive

SW#show vtp status

Display VTP status, including overview

SW#show vtp counters

Show Switch VTP Counters

SW(config-if)#switchport mode trunk

Changing the Interface Mode in Trunk

SW(config)#vlan 10

Create a VLAN and associate it with ID 10

SW(config-vlan)#name sales

Name the VLAN

SW(config)#interface fastethernet 0/1

Enter Interface Configuration Mode

SW(config-if)#switchport mode Access

Set the interface binding type to access it

SW(config-if)#switchport Access vlan 10

Asignar interface a VLAN 10

SW#show vlan

Show VLAN Information

SW#show vlan brief

Show short VLAN information

SW#show vlan id 10

Show ID 10 Only VLAN Information

SW#show vlan name HR

Show information only for the named VLAN HR

SW(config)#interface fastethernet 0/8

Enter Interface Configuration Mode

SW(config-if)#no switchport Access vlan 10

Remove the interface from VLAN 10 and reassign it to VLAN 1 by default

SW(config)#no vlan 10

Delete VLAN 10 from the VLAN database
